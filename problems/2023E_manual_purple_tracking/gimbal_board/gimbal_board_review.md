# 云台小板资料审查

## 找到的资料

- `DCC-100v1v2说明书.pdf`、`DCC-100v3说明书-2026-05-24.pdf`。
- MSPM0G3507 示例：基础转动、速度/角度控制、`【云台】08_按键上下左右遥控开环电机`。
- 两版 STM32F103C8T6 参考工程及 `motor.c/h`。

## 硬件性质与接口结论

DCC-100 是双路步进电机驱动模块。说明书照片和接口表只给出每路 `PWM(步进脉冲)、DIR、DCY、SLP、RST、GND`；逻辑电平 3.3V，电机侧电源 11-26V，一脉冲对应一个微步（默认 1/32 时资料标称约 0.05625 度）。没有发现板载 MCU、UART、I2C 或可解析 `$GM` 的协议。

MSPM0 与 STM32F103C8T6 工程均由外部 MCU 直接产生脉冲并控制 DIR。可见函数包括 `step_motor_dir_set/start/stop/set_speed/set_angle` 和 `Motor_SetSpeed/Start/Stop/SetDirection/Turn_Angle`。这些是真实执行代码且缺少已验证的闭环限位/回零协议，本阶段不复制、不调用。

## 推荐接入

当前不能把 DCC-100 当成智能“云台专用小板”，也不能向它发送 UART `$GM`。推荐保持：ZET6 只生成抽象 pan/tilt 与 dry-run 字符串；待用户确认是否还有一块位于 ZET6 与 DCC-100 之间、带 MCU 的控制小板及其固件/接口后，再决定 UART/I2C。

如果最终确认没有中间 MCU，DCC-100 实际是无协议驱动板，届时需要另立安全任务评审“由哪个 MCU 产生 STEP/DIR”，不能在本任务中直接启用 PWM。GPIO 直连也不足以安全运行，因为还需要脉冲定时、限位、回零、速度/加速度和故障停机。

## 不确定项与接线前确认

1. 实物版本是 V1/V2 还是 V3，以及两轴电机额定电流/电压。
2. 是否确有另一块带 MCU 的云台控制小板；若有，请提供正反面照片、芯片丝印、接口图和固件。
3. 两轴机械方向、减速比、行程、零位与限位开关情况。
4. DCC-100 的 LC/HC 电流设置、DCY/SLP/RST 默认状态和急停断电路径。
5. ZET6 与任何底层板的逻辑电平、共地、隔离和供电拓扑。

在以上信息确认前，推荐接口结论为“暂时无法确定”；UART 仅是未来智能中间板的优先候选，DCC-100 本体则是 STEP/DIR 类低层驱动。
