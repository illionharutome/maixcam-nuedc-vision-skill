# ZET6 到 C8T6 分阶段接入计划

总体串口规划：USART1 保留为 PC/ZET6 debug bridge，继续接收模拟 `$MV,AIM/$MV,TRACK1` 并回传 `$DBG`。未来 USART2 或 USART3 才作为 ZET6 -> C8T6 通道。当前没有已确认的 C8T6 协议，所有 `$GM,CMD` 仅为占位 dry-run 格式。

## 阶段 0：资料审查，不接线

- 动作：确认 C8T6 PCB/固件版本、电源、UART 可用引脚、电机/DCC-100、限位和急停。
- 退出条件：形成经人工确认的引脚表、电源图和“上电默认禁能”方案。
- 风险：错认版本或把 DCC-100 当智能协议板。
- 安全：所有电机电源断开，保留可见的总断电方式。

## 阶段 1：只生成命令字符串

- 动作：ZET6 内存中调用 `gimbal_board_build_command()`，Keil Watch 观察 `$GM,CMD,...#`，不初始化第二串口。
- 退出条件：边界值、无效 mode、NO_TARGET/LOST/ERROR/AIMED 归零测试通过。
- 风险：将抽象毫单位误认为角度、速度或脉冲。
- 安全：无 C8T6、DCC-100 和电机接线；断电方式保持可用。

## 阶段 2：第二串口只接 USB-TTL（已实现）

固件侧：`USER/bridge_config.h` 中 `BRIDGE_ENABLE_GIMBAL_DRY_UART=0`（仓库默认），
USART3_TX (PB10) → USB-TTL RX，代码已就绪。

> 接线前核对板卡原理图/丝印，确认 PB10 未被其他外设占用。

测试脚本：`problems/2023E_manual_purple_tracking/scripts/stm32_gimbal_dry_uart_check.py`

用法：
```text
python problems\2023E_manual_purple_tracking\scripts\stm32_gimbal_dry_uart_check.py --debug-port COM5 --gimbal-port COM7
```

- 动作：用户本地改 `BRIDGE_ENABLE_GIMBAL_DRY_UART` 为 1，Build，烧录；
  PB10 接 USB-TTL RX，GND 共地；PC 串口助手或双串口脚本观察 `$GM,CMD`。
  C8T6 不接线。
- 退出条件：TRACKING/AIMED/STOP mode 输出正确，USART1 AIM/TRACK1 回归继续 PASS。
- 风险：选错引脚/电平、串口资源冲突、消息风暴。
- 安全：USB-TTL 与 ZET6 正确共地，电机电源保持断开，有立即拔线/断电方式。

## 阶段 3：连接 C8T6，不接电机

- 前提：C8T6 固件已在单独审查中改为上电禁能、只解析命令且有超时停机；当前原始往返演示固件禁止使用。
- 动作：ZET6 TX 接 C8T6 RX，必要时接回传；DCC-100 和电机均不接。通过 C8T6 Watch/ACK 验证接收，不调用 Motor_Start。
- 退出条件：合法/非法/截断/超时命令均符合安全状态，断开 ZET6 后 C8T6 保持禁能。
- 风险：原固件上电自动运行、TX/RX 或电平错误、未共地。
- 安全：不上电机电源，MCU 供电设限流，保留两端断电。

## 阶段 4：空载低速电机测试

- 前提：阶段 3 通过，并完成限位、方向、最大步频、加速度、DCC 电流和急停专项审查。
- 动作：单轴、最低安全电流和速度、极短脉冲窗口测试；另一轴断开或禁能。
- 退出条件：方向、停止、通信超时、急停、限位和温升均通过，再逐轴重复。
- 风险：失控旋转、夹伤、过流、堵转、反向和机械撞限。
- 安全：空载固定、人员远离旋转件、物理急停/总电源开关在手边，先断电再改线。

## 阶段 5：上云台载荷测试

- 前提：阶段 4 全部通过，机械结构、线束、载荷、软硬限位和光源安全完成审查。
- 动作：低速、小角度、单轴开始，再做双轴；逐级提高但不得越过批准上限。
- 退出条件：追踪方向、稳定性、温升、线缆余量、失目标归零和急停均满足记录标准。
- 风险：载荷跌落、线缆缠绕、共振、撞限、光束风险。
- 安全：固定屏幕、物理遮挡、禁止高功率/405nm，急停与总断电始终由专人控制。

任何阶段出现协议不确定、异常动作、通信丢失、温升、噪声复位或限位失效，都立即退出到上一阶段，不允许自动升级阶段。
