# YOLO 后续训练与部署计划

YOLO 不是当前 2025E 红点/固定靶心闭环的依赖。传统 LAB 阈值和几何方法可稳定解决时，不增加模型复杂度。

只在目标类别复杂、背景变化大、传统方法不稳定、需要识别特定物体/标志/零件/障碍物/抓取目标，或题目明确要求类别识别时考虑 YOLO。

未来流程：

1. 按场景采集图像，记录光照、距离、背景、相机参数和任务类型。
2. 制定类别与框标注规范，复核漏标、错标和模糊样本。
3. 按场景而非随机相邻帧划分训练、验证和独立测试集，避免数据泄漏。
4. 记录模型版本、输入尺寸、增强、训练参数和数据集版本。
5. 评估每类 precision、recall、mAP、误检、漏检、端侧 FPS 和内存，不只看总 mAP。
6. 经验证后导出 ONNX，再按当时 MaixPy/MaixCAM 官方流程转换和部署；当前不执行转换。
7. 用适配器把输出转换为 `VisionResult` 或未来 `TaskResult`，不能绕过统一协议。

传统视觉和 YOLO 应作为可选择的检测后端共存；2025E 默认仍走传统视觉。
